# NATHub 网络设计

## 1. 网络前提

当前网络环境中：

- MT7621 可访问 OpenWrt / pfSense / Huawei SRG 设备
- 控制器部署在 x86 环境
- 不希望在 MT7621 上部署任何业务程序
- 已确认可完成三层路由打通

因此推荐模式为：

> x86 控制器通过三层路由经 MT7621 访问目标设备管理地址

---

## 3. 设计原则

### 3.1 MT7621 只做网络设备
MT7621 不承担任何控制器业务逻辑，仅负责：

- 路由转发
- VLAN
- ACL
- 必要时 SNAT

### 3.2 x86 直连目标设备
控制器程序直接连接：

- OpenWrt：SSH
- pfSense：HTTP API / HTTPS
- Huawei SRG：SSH CLI

### 3.3 设备回程必须正确
目标设备需要能正确回到控制器地址。

实现方式二选一：

- 为控制器网段添加静态路由
- 或在 MT7621 上对管理流量做 SNAT

---

## 4. 两种可达实现

### 4.1 方案 A：纯三层路由
优点：

- 最清晰
- 最易排障
- 程序实现最简单

要求：

- MT7621 有正确转发表
- 目标设备有回程路由

### 4.2 方案 B：管理流量 SNAT
优点：

- 不需要在每台设备上加回程路由
- 适合复杂或老旧环境

缺点：

- 设备侧看到的源地址是 MT7621
- 审计粒度较粗

---

## 5. 网络访问建议

### OpenWrt
- SSH 22
- 可选 HTTPS Web 管理口

### pfSense
- HTTPS API / Web 管理端口
- 可选 SSH

### Huawei SRG
- SSH 22

---

## 6. 安全建议

- 控制器到设备管理地址仅开放必要端口
- 限制来源为控制器管理 IP
- 单独划分管理 VLAN / 管理网段
- 不开放目标设备管理地址到普通业务网段

# NATHub 架构设计

## 1. 项目目标

NATHub 是一个统一 NAT / 端口映射管理平台，运行在 x86/Linux 控制器上，用于集中管理多设备、多出口场景下的端口映射规则。

目标支持设备：

- OpenWrt
- pfSense
- Huawei SRG2210

核心目标：

- 统一管理不同厂商设备的 NAT / 端口映射
- 支持多设备、多出口、多 WAN
- 为 PVE 内部主机提供集中映射管理
- 提供规则统一抽象、下发、查询、删除、同步能力
- 提供审计日志与任务执行记录

---

## 2. 总体架构

采用 **x86 单控制器架构**：

- 控制器部署在 PVE VM 或独立 x86 Linux 主机
- 控制器通过三层路由直连目标设备管理地址
- MT7621 仅提供路由可达性，不部署 agent 或业务程序
- 所有设备适配逻辑、规则编排、审计、任务执行都集中在控制器完成

---

## 3. 架构原则

### 3.1 单控制器
所有业务逻辑在一个 Python 程序内完成，降低维护复杂度。

### 3.2 网络直连
通过三层路由让控制器直接访问目标设备，避免在边界设备上增加代理程序。

### 3.3 统一规则模型
先定义平台统一 NAT 规则模型，再映射到不同设备配置形式。

### 3.4 适配器分层
每类设备使用独立 Adapter，屏蔽厂商差异。

### 3.5 传输层分离
设备连接方式（SSH / HTTP API）与设备能力实现解耦。

---

## 4. 分层设计

### 4.1 Web 管理层
负责：

- 登录认证
- 页面展示
- 表单录入
- 提供 API

### 4.2 业务服务层
负责：

- NAT 规则校验
- 冲突检测
- 任务调度
- 状态同步

### 4.3 设备抽象层
负责定义统一设备能力接口：

- 查询 NAT 规则
- 创建 NAT 规则
- 删除 NAT 规则
- 更新 NAT 规则
- 测试连通性

### 4.4 厂商适配层
包括：

- OpenWrtAdapter
- PfSenseAdapter
- HuaweiSRGAdapter

### 4.5 传输层
包括：

- SSHTransport
- HTTPTransport

### 4.6 数据存储层
负责保存：

- 设备信息
- 出口信息
- NAT 规则
- 执行任务
- 审计日志

---

## 5. 典型数据流

### 新建规则
1. 用户在 Web 界面填写 NAT 规则
2. 后端校验参数合法性
3. 检查数据库中是否冲突
4. 调用设备适配器读取当前设备规则
5. 再次检查设备侧冲突
6. 生成任务并执行下发
7. 保存结果与日志
8. 更新规则同步状态

### 删除规则
1. 用户请求删除规则
2. 后端生成删除任务
3. 调用设备适配器删除目标规则
4. 记录执行结果和审计日志

---

## 6. 首期范围

MVP 首期只覆盖：

- 设备管理
- 出口管理
- NAT 规则 CRUD
- OpenWrt 规则下发
- 基础任务日志
- 基础审计日志

pfSense 和 Huawei SRG 作为后续阶段逐步接入。
